Programação Web Avançada — Ficha GitHub

1 - O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?

Commit: guarda uma versão local das alterações.

Push: envia as alterações locais para o repositório remoto.

Pop: reaplica alterações temporárias guardadas com stash.

  • Commit: Local
  • Push: Remoto
  • Pop: Recupera mudanças guardadas

2 – Quais são as vantagens de usar Branchs num repositório git?

  1. Permite desenvolver novas funcionalidades em paralelo.
  2. Evita alterar o código principal (main).
  3. Facilita trabalho colaborativo e revisão de código.

3 – Nos Merges resultantes de um “Pop” podem ocorrer conflitos. Em que consistem e como se resolvem?

Conflitos surgem quando duas alterações interferem entre si. O Git marca as diferenças no ficheiro.

  • Editar o ficheiro e escolher as linhas corretas.
  • Guardar e usar git add + git commit.

4 – “Clone” e “Fork” replicam repositórios Git. Quais as diferenças?

Clone: copia um repositório localmente no teu computador.

Fork: cria uma cópia do repositório na tua conta GitHub, para modificações independentes.
